KEY PRODUCTS AT A GLANCE

🐧 openEuler OS

  • Huawei's open-source Linux distribution
  • Version used: 20.03 LTS (HCIA/HCIP) and 22.03 LTS (HCIE-Ceph)
  • Package manager: DNF / RPM
  • Init system: systemd
  • Supports x86_64 and ARM (Kunpeng) architectures

🗄️ openGauss Database

  • Huawei's open-source relational database (PostgreSQL-based)
  • Version used in labs: V2.0.0 (HCIA), V5.0.0 (HCIP/HCIE)
  • Enterprise edition used for HCIP and HCIE
  • CLI client: gsql
  • Also accessible via DAS (Data Admin Service) on Huawei Cloud
  • Managed cloud version: GaussDB

HCIA-openEuler: 9 Core Lab Topics

HCIA

Lab 1: Installing openEuler OS

  • Install VirtualBox and create a VM
  • Mount ISO: openEuler-20.03-LTS-x86_64-dvd.iso
  • Network adapter: set to Bridged Adapter
  • Partition layout: /boot/efi (200MB), /boot (200MB), /swap (4GB), / (rest)
  • Enable CPU virtualization in BIOS first
  • Use PuTTY for remote SSH access
HCIA

Lab 2: Basic CLI Operations

  • bash command basics and shell navigation
  • File viewing: cat, less, head, tail
  • Packaging: tar, gzip, zip
  • Help: man, --help, info
  • Directory navigation: cd, ls, pwd
HCIA

Lab 3: Vim Text Editor

  • 3 modes: Normal, Insert, Command
  • Move: h (left) j (down) k (up) l (right)
  • Enter insert: i (before), a (after), o (new line below)
  • Exit: :q quit, :w save, :wq save+quit, :q! force quit
  • Delete: x (char), dd (line), d$ (to end)
  • Search: /pattern forward, ?pattern backward
  • Replace all: :%s/old/new/g
  • Undo: u, Redo: Ctrl+R
HCIA

Lab 4: User & Permission Management

  • Add user: useradd username
  • Set password: passwd username
  • Delete user: userdel -r username
  • Permission format: rwxrwxrwx (owner/group/others)
  • Change permissions: chmod 755 file or symbolic chmod u+x file
  • Change owner: chown user:group file
  • Switch user: su - username
  • Sudo privileges: edit /etc/sudoers
HCIA

Lab 5: Software Management

  • Configure Yum/DNF repo in /etc/yum.repos.d/
  • DNF: dnf install -y pkg, dnf remove pkg, dnf list
  • RPM: rpm -ivh pkg.rpm (install), rpm -qa (query all), rpm -e pkg (erase)
  • RPM options: -v (verbose), -h (hash), -U (upgrade)
  • Practical: Install Nextcloud with Apache + PHP
HCIA

Lab 6: Storage & File System

  • Add a new disk; view with lsblk, fdisk -l
  • MBR partitioning: fdisk — primary, extended, logical
  • GPT partitioning: parted — interactive and non-interactive
  • Format: mkfs.ext4 /dev/sdb1 or mkfs.xfs
  • Mount: mount /dev/sdb1 /mnt/data
  • Persist mount: edit /etc/fstab
  • LVM: PV → VG → LV → Format → Mount
  • Expand LV: lvextend + resize2fs
HCIA

Lab 7: System & Process Management

  • Task scheduling: at (one-time), crontab -e (recurring)
  • cron syntax: min hr dom mon dow command
  • Process: ps aux, top, kill PID, kill -9 PID
  • Network: ip addr, ip route, ping, ss -tuln
  • Hostname: hostnamectl set-hostname name
  • Firewall: firewall-cmd --add-service=http --permanent
  • Service mgmt: systemctl start/stop/enable/status svc
HCIA

Lab 8: Shell Scripting

  • Variables: name="value", access with $name
  • Positional params: $1, $2 ... $@ all, $# count
  • if: if [ condition ]; then ... fi
  • while: while [ cond ]; do ... done
  • for: for i in list; do ... done
  • case: case $var in pattern) ...;; esac
  • Test: -eq equal, -ne not equal, -lt less than, -f file exists

HCIP-openEuler: Enterprise Services

HCIP

Lab 1 & 2: Apache & NGINX

  • Install Apache: dnf install -y httpd
  • Apache version: 2.4.51
  • Start service: systemctl enable httpd --now
  • Config file: /etc/httpd/conf/httpd.conf
  • Set ServerName in line 97 to suppress errors
  • NGINX: static resources, virtual hosts, location directive
  • NGINX reverse proxy & load balancing config
  • Configure HTTPS/SSL for Apache
  • MPM working modes: prefork, worker, event
  • Apache DSO module load/unload
HCIP

Lab 3 & 4: DNS & MySQL

  • DNS software: BIND, install and configure
  • Set up master DNS server with forward zone
  • Configure reverse DNS resolution
  • MySQL: dnf install mysql-server
  • Initialize: mysqld --initialize
  • Login: mysql -u root -p
  • Use mysqladmin for admin tasks
  • CRUD: INSERT, SELECT, UPDATE, DELETE
HCIP

Lab 5: LAMP Stack (Comprehensive)

  • LAMP = Linux + Apache + MySQL + PHP
  • Connect Apache with PHP module
  • Connect PHP with MySQL (PDO/mysqli)
  • Deploy WordPress as the capstone application
  • Configure 4 ECS servers: Apache (S01), NGINX (S02), DNS (S03), MySQL (S04)

HCIE-openEuler: Ceph Cloud Storage (7 Labs)

HCIE

Lab 1: Installing Ceph

  • 3 Ceph nodes (ceph01–03) + 1 Client node on openEuler 22.03 LTS
  • Ceph nodes: each with three 10GB data disks as OSDs
  • Pre-steps: hostname resolution, disable firewall & SELinux, time sync
  • Tool: Cephadm for bootstrapping and managing Ceph
  • Install container engine (Podman) before Ceph
  • Bootstrap: cephadm bootstrap --mon-ip <IP>
  • Add hosts and OSDs after bootstrap
  • Operations: add/remove services, OSDs, nodes
HCIE

Lab 2: Ceph Resource Pool Management

  • Create replicated pool (data copied N times)
  • Create erasure-coded pool (more efficient for large data)
  • Set replica count and specifications
  • Create pool snapshots
  • Key command: ceph osd pool create <name> <pg_num>
HCIE

Lab 3: Ceph User Management

  • Create users and grant permissions (mon, osd, mds)
  • Export and import authorization (keyring) files
  • ceph auth get-or-create to create users
  • ceph auth export / ceph auth import
HCIE

Lab 4: Ceph Object Storage (RGW)

  • Create RADOS Gateway (RGW) — S3-compatible object store
  • Configure object bucket quotas
  • Multi-zone disaster recovery (DR) configuration
  • Access via S3-compatible clients (e.g., s3cmd)
HCIE

Lab 5: Ceph Block Storage (RBD)

  • Create RADOS Block Devices (RBD)
  • Map and mount RBD to client as a block device
  • Expand (extend) RBD capacity online
  • Create, delete, and restore RBD snapshots
  • rbd create, rbd map, rbd snap create
HCIE

Lab 6 & 7: CephFS & CRUSH Map

  • CephFS: Create file storage pool and MDS service
  • Mount CephFS to client: mount -t ceph ...
  • CRUSH Map: Controls data placement in the cluster
  • Manually edit CRUSH map to change drive type to SSD
  • Compile/decompile CRUSH map: crushtool

HCIP-openGauss: 6 Advanced Labs

HCIP

Lab 1: Advanced Object Management

  • System catalogs: pg_class, pg_attribute, pg_namespace
  • System views: pg_stat_activity, dv_session
  • View/stop running queries (pg_terminate_backend)
  • Resource monitoring views
  • Grant permissions: GRANT SELECT ON table TO user
  • ANY permissions (system-level)
  • Scheduled jobs: DBMS_JOB or PKG_SERVICE package
HCIP

Lab 2: Database System Management

  • Routine health checks: cluster status, OS check
  • Command: gs_om -t status --detail
  • Check DB parameters in postgresql.conf
  • Monitor lock waits
  • WDR Report: Workload Diagnosis Report for performance snapshots
  • Generate static config files with gs_om
HCIP

Lab 3: Advanced SQL

  • Complex JOINs: INNER, LEFT, RIGHT, FULL OUTER, CROSS
  • UNION, UNION ALL, INTERSECT, EXCEPT
  • Subqueries: IN, ALL, ANY, EXISTS, correlated
  • Advanced functions: numeric, string, date, type conversion, system info
  • Stored Procedures: PL/pgSQL syntax, parameters, loops
  • Triggers: BEFORE/AFTER INSERT/UPDATE/DELETE
  • API connectivity: JDBC, ODBC, Python (psycopg2)
HCIP

Lab 4: Database Migration

  • Tool: DataKit (Huawei's migration platform)
  • Migrate from MySQL → openGauss
  • Steps: deploy DataKit → create data sources → add server → preset MySQL data → create and run migration task
  • Install migration plug-ins within DataKit
  • JDK and MySQL must be installed first
HCIP

Lab 5: Database Security

  • SSL: Set up CA, create server & client certificates
  • Configure SSL in postgresql.conf and pg_hba.conf
  • User/role permission management (GRANT/REVOKE)
  • Row-Level Security (RLS): control access per row
  • Security Audit: unified audit logging
  • Encrypted Equality Query: query on encrypted data
  • Ledger Database: tamper-proof history tables

HCIE-openGauss: 8 Expert Labs

HCIE

Lab 1: Transactions & Locks

  • ACID properties: Atomicity, Consistency, Isolation, Durability
  • Auto-commit mode; explicit transactions: BEGIN/COMMIT/ROLLBACK
  • Isolation levels: READ COMMITTED, REPEATABLE READ
  • Subtransactions: SAVEPOINT … ROLLBACK TO SAVEPOINT
  • Lock types: SHARED, ACCESS SHARE, ROW EXCLUSIVE, ACCESS EXCLUSIVE
  • View locks: SELECT * FROM pg_locks;
HCIE

Lab 2: Storage Management

  • Compile openGauss from source code
  • Install pageinspect plugin to inspect data pages
  • Analyze page header structure
  • View tuple data and raw data on a page
  • Understand row-level storage layout
HCIE

Lab 3: WAL Log Management

  • WAL = Write-Ahead Logging (crash recovery mechanism)
  • View WAL files in pg_wal directory
  • Parse WAL files with pg_waldump
  • Auto-restore database using WAL after crash
  • PITR: Point-In-Time Recovery using WAL + base backup
  • Check and clear system logs
HCIE

Lab 4: SQL Execution Engine

  • Scan operators: Seq Scan, Index Scan, TID Scan, Function Scan
  • Join operators: Hash Join, Merge Join, Nested Loop Join
  • Control operators: Result, Limit, Append
  • Materialization: Sort, Hash, Unique, Aggregate, Group, SetOp
  • Read EXPLAIN plans to understand query execution
  • EXPLAIN ANALYZE for actual execution stats
HCIE

Lab 5: Database Performance Optimization

  • Check server hardware: CPU, memory, OS version
  • Monitor CPU load and memory usage
  • Static optimization: indexes, table partitioning, vacuum
  • Dynamic optimization: query rewrites, execution plan hints
  • Use system views for real-time monitoring
HCIE

Lab 6: Backup, Restore & DR

  • gs_dump / gs_dumpall for logical backups
  • gs_basebackup for physical base backups
  • PITR: restore to a specific point in time
  • Flashback: recover accidentally deleted data
  • DR setup: primary/standby replication
HCIE

Lab 7: Advanced Features

  • AI capabilities: built-in ML features in openGauss
  • Fully-encrypted databases: data encrypted at rest and in use
  • MOT (Memory-Optimized Tables): in-memory OLTP performance
HCIE

Lab 8: Troubleshooting

  • Diagnose common database errors from logs
  • Resolve connection failures, lock deadlocks
  • Handle performance degradation issues
  • Check system logs: /var/log/ and openGauss log path
